So we use postman to test our API about burgers. We start with our get request.

So as espected we get out data from our in-memory database, back as JSON object


Next we make a post request to add a new burger to our database

So we can see we added our new burger to the database, we get a response back with the burger we added. so if we make a new get request we will see the new burger added with an incrementet id also.


Let's say right after we added the burger, its not the correct burger. Then we can use patch to change the burger, in the url we choose witch burger we want to change.

we just need to use the same key, like "name" and "price" in this case and just change the value we want to change.


The last request is our delete request. Where you just delete the burger on the id, through the url paramenter

So again if we run the get request, we will se that one burger is delete depending on the id. In this case, burger 1 will be deleted.
